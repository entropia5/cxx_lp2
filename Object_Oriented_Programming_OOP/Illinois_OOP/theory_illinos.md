If-Else

В приведенном выше примере также показана одна из наиболее фундаментальных управляющих структур в C++: Ключевые слова if и else. Базовая структура всегда выглядит следующим образом:

if (condition) {
    // true case
}
else {
    // false case
}



Можно выстроить цепочку из нескольких комбинаций if-else. В этой ситуации первое условие, которое окажется истинным, будет той секцией, которая будет выполнена:

if (condition1) {

}
else if (condition2) {

}
else if (condition3) {

}
else {
    // If none of the other conditions were met...
}




Очень важно отметить, что в приведенном выше примере использование "если... else если... else" в цепочке делает условные случаи взаимоисключающими. Выполнится только один из них: первый, который окажется истинным. Будьте очень внимательны, используя "else" там, где это уместно, чтобы добиться такого поведения. Если Вы просто поместите несколько операторов "if" в последовательность, то потенциально несколько из них будут выполнены, как в этом примере:

int x = 100;
if (x > 0) {
    // This will execute
}
if (x > 10) {
    // This will execute
}
if (x > 50) {
    // This will execute
}
else {
    // This is the only section that won't execute!
}

Здесь все условные блоки будут выполнены, за исключением последней части "else": последнее "else" в конце влияет только на предыдущее "if", и поскольку верно, что x больше 50, этот блок "if" будет выполнен, а последнее "else" - нет. Важный урок заключается в том, что всегда нужно тщательно выбирать, где писать "else" (или не писать), в зависимости от того, хотите ли Вы, чтобы случаи были взаимоисключающими.

Вы также должны знать о странном на вид "тернарном операторе", называемом также "условным оператором", который на самом деле представляет собой комбинацию двух операторов, "?" и ":" (вопросительный знак и двоеточие). Основной формат этого синтаксиса следующий:

[условие с булевым значением] ? [выражение, которое нужно оценить, если оно истинно] : [выражение, которое нужно оценить, если оно ложно]

Таким образом, Вы можете сравнить "A ? B : C" с "if (A) {B;} else {C;}", но тернарный оператор разрешен в ситуациях, когда "if" не разрешен. Это объясняется тем, что оператор "?:" оценивается на уровне выражения, поэтому он может быть подвыражением внутри большего выражения, в то время как оператор "if" является оператором управления потоком верхнего уровня, который не может быть вложен в выражение. Это означает, что Вы можете поместить ?: справа от оператора присваивания =, чего нельзя сделать с оператором "if". Вот пример:


// Since (5<10) is true, the expression before the colon will be selected, which is 1.
int x = 5 < 10 ? 1 : 2;
// Now x is equal to 1.

// Note, the following syntax is NOT allowed in C++, which is why the ternary operator can be useful in these cases:
int y = if (5<10) {1;} else {2;}



Этот троичный оператор ?: полезен и в других ситуациях, когда Вы можете использовать его творчески, чтобы сделать код более кратким и компактным, хотя некоторые люди находят его запутанным для чтения. Мы рекомендуем Вам не использовать его чаще, чем это необходимо, но Вы будете периодически видеть его при изучении исходного кода на языке Си++.

Приведение типов

Важной концепцией, которую необходимо понять в C++, является "приведение" Это явление, при котором значение автоматически преобразуется к другому типу для того, чтобы операция могла быть продолжена. Например, если Вы попытаетесь сложить значение double с плавающей точкой и целочисленное значение int , то сначала int будет преобразовано в double, а затем результат сложения будет иметь тип double. Но если Вы сохраните значение double в переменной int, то значение с плавающей точкой будет усечено до целого числа! Например:


#include <iostream>
int main() {
  int x = 2;
  double y = 3.5;
  std::cout << "This result will be a double with value 5.5: " << (x+y) << std::endl;

  int z = x+y; // This expression is calculated as a double, but then it's cast back to int!
  std::cout << "This result will be an int with value 5: " << z << std::endl;
  return 0;
}



Различные значения могут быть приведены к булевому типу bool . Таким образом, они могут быть использованы в качестве условия. Например, обычно ненулевые числовые значения будут считаться истинными, а нулевые - ложными.

#include <iostream>
int main() {
  if (0) {
    std::cout << "You won't see this text." << std::endl;
  }
  if (-1) {
    std::cout << "You WILL see this text!" << std::endl;
  }
  return 0;
}


Заметки по синтаксису C++: Область действия блока, циклы
Область видимости блока

После просмотра лекции о стековой памяти у Вас уже должно быть интуитивное представление о том, как работает область видимости блока. Идея заключается в том, что определенные блоки кода, обозначаемые символами { }, создают внутренний стек поверх предыдущего стека, который может скрывать ранее существовавшие значения. Время жизни стековых переменных внутри блока называется областью видимости переменной. Переменные, созданные на стеке внутри внутреннего блока, существуют только в течение всего времени работы блока. Когда блок заканчивается, внутренний стек удаляется, и ранее существовавшие значения во внешней области видимости снова становятся доступными (потому что они все еще находятся в области видимости!).

Вот пример:

#include <iostream>
int main() {

  // You can actually make an inner scope block anywhere in a function.
  // Let's do it to show how scope works.

  // In the initial, outer scope:
  int x = 2;
  std::cout << "Outer scope value of x (should be 2): " << x << std::endl;

  // Create an inner scope and make a new variable with the name "x".
  // This is not an error! We can redeclare x because of the inner scope.
  // Also, make an extra variable named "y".
  {
    int x = 3;
    int y = 4;
    std::cout << "Inner scope vaue of x (should be 3): " << x << std::endl;
    std::cout << "Inner scope vaue of y (should be 4): " << y << std::endl;
  }

  // Now that the inner block has closed, the inner x and y are gone.
  // The original x variable is still on the stack, and it has its old value:
  std::cout << "Outer scope value of x (should be 2): " << x << std::endl;

  // We can't refer to y here, because it doesn't exist in this scope at all!
  // If you un-comment this line, there will be a compile error.
  // std::cout << "This line causes an error because y doesn't exist: " << y << std::endl;

  return 0;
}




Некоторые ключевые слова, такие как if , могут иметь после себя блок в { }, который действительно создает внутреннюю область видимости блока на время действия условного блока:

#include <iostream>
int main() {
  int x = 2;
  std::cout << "Outer scope value of x (should be 2): " << x << std::endl;

  if (true) {
    int x = 3;
    std::cout << "Inner scope vaue of x (should be 3): " << x << std::endl;
  }

  std::cout << "Outer scope value of x (should be 2): " << x << std::endl;

  return 0;
}

Циклы

В Си++ существует несколько видов циклов, которые позволяют Вам итеративно обрабатывать данные. Здесь мы просто покажем Вам несколько типов.

циклы for

Цикл for - это распространенный тип цикла, который позволяет Вам указать итерационную переменную, диапазон для этой переменной и инструкцию инкремента. Синтаксис выглядит следующим образом:

for ( объявление ; условие; операция инкремента ) { тело цикла }

Будьте внимательны к тому, объявляете ли Вы заново итерационную переменную в области видимости блока или нет! Вот пример:


#include <iostream>
int main() {

  // outer scope version of "x" to show the for-loop block scoping
  int x = -1;

  std::cout << "[Outside the loop] x is now (should be -1): " << x << std::endl;

  // The for loop lets us declare a variable in the first part of the
  // loop statement, which will belong to the inner block scope:
  for (int x = 0; x <= 2; x++) {
    std::cout << "[Inside the loop] x is now: " << x << std::endl;
  }

  // The outer scope x is still -1
  std::cout << "[Outside the loop] x is now (should be -1): " << x << std::endl;

  // This version doesn't redeclare x, so it just inherits access to the
  // same x variable from the outer scope. This modifies the outer x directly!
  for (x = 0; x <= 2; x++) {
    std::cout << "[Inside the loop] x is now: " << x << std::endl;
  }

  // In the last iteration where the condition x<=2 was true, x had the value 2.
  // After that iteration, x was incremented one more time and became 3.
  // Then the condition was false and the loop body did not execute.
  // Afterwards, the outer scope x is still 3 because the loop modified it.
  std::cout << "[Outside the loop] x is now (should be 3): " << x << std::endl;

  return 0;
}



циклы while

Еще одним циклом является простой цикл while, который имеет такой синтаксис:

while ( condition ) { тело цикла }




#include <iostream>
int main() {
  int x = 0;
  std::cout << "This should show 0, 1, 2, 3:" << std::endl;
  while (x <= 3) {
    std::cout << "x is now: " << x << std::endl;
    x++; // increment x by 1
  }
  return 0;
}



1. Что такое конструктор?
Это специальная функция-член класса, которая вызывается автоматически в момент создания объекта. Ее главная задача — привести объект в рабочее состояние (например, установить длину сторон куба).

Три правила конструктора:
Название функции совпадает с названием класса (Cube).
У функции нет типа возвращаемого значения (даже void).
Она вызывается автоматически.


2. Автоматический vs Специальный (Custom)

Автоматический конструктор за умолчанию: C++ дает его тебе «в подарок», если ты не написал вообще ни одного своего конструктора. Он просто создаст объект, но переменные (как length) будут содержать «мусор».
Специальный (Custom) конструктор: Это тот, который написал ты сам.
Золотое правило: Как только ты написал хотя бы один любой свой конструктор (даже если он принимает 10 параметров), «подарочный» автоматический конструктор исчезает.

3. Типы конструкторов на примере Cube

Это конструктор, который не принимает аргументов.

Программист написал конструктор Cube(double length).

Когда Вы захотите использовать память кучи, Вы воспользуетесь оператором "new". Как и в случае со стековой памятью, Вы должны быть осторожны, когда используете "new" для встроенного типа, такого как int, поскольку эти типы могут не иметь инициализации по умолчанию. Поэтому Вы не должны предполагать, что они будут иметь ожидаемое значение по умолчанию (например, 0). В этих случаях обязательно инициализируйте значение вручную. Вот несколько примеров.

Указатель "q" создается и инициализируется адресом только что выделенного целого числа на куче. Однако целое число, на которое указывает q, не имеет предсказуемого значения. Оно зависит от компилятора. Мы не должны полагаться на то, что это целое число будет иметь какое-то конкретное значение в начале.

int* q = new int;



Вы можете указать параметры инициализации в конце выражения "new". В результате будет создан указатель "r", инициализированный вновь выделенной памятью для целого числа со значением "0" в явном виде.

int* r = new int(0);



Сброс удаленных указателей на nullptr

Теперь, когда мы рассмотрели инициализацию, особенно для указателей, давайте поговорим о том, почему Вам следует вручную сбрасывать указатели в nullptr, когда Вы с ними закончили работу.

Обратите внимание, что использование команды "delete" для указателя освобождает память кучи, выделенную по этому адресу. Однако удаление указателя не приводит к автоматическому изменению значения указателя на "nullptr"; Вы должны сделать это сами после использования "delete" в качестве меры предосторожности. Например:

// Allocate an integer on the heap:
int* x = new int;
// Now x holds some memory address to a valid integer.
// Do some kind of work with the integer.
// We'll just set that integer to 7:
*x = 7;
// Now delete the pointer to deallocate the heap memory:
delete x;
// This destroys the integer on the heap and frees the memory.
// But now x still holds the memory address!
// Set x to nullptr for safety:
x = nullptr;


Идея здесь заключается в том, что, устанавливая вручную значение x в nullptr после "delete x", мы помогаем предотвратить два вида проблем:

1. Мы не хотим по ошибке удалять один и тот же выделенный адрес более одного раза, что может привести к ошибкам. Использование "delete" для nullptr ничего не дает, так что в этом случае, если мы случайно попытаемся удалить один и тот же адрес дважды, ничего не произойдет.

2. Мы никогда не должны разыменовывать указатель на деаллоцированную память. Это может привести к аварийному завершению программы с ошибкой сегментации, странному поведению или уязвимости в системе безопасности. Однако попытка разыменовать nullptr почти всегда вызовет segfault и немедленное завершение программы. Это на самом деле лучше, чем вызвать "тихую" уязвимость безопасности или другую проблему, которую труднее обнаружить! Поэтому имеет смысл установить для удаляемого указателя значение nullptr, гарантируя тем самым, что если мы по неосторожности разыменуем его, то это вызовет очень очевидную ошибку времени выполнения, которую мы сможем исправить.

Следует также отметить, что использовать delete и nullptr нужно, конечно, только с указателями. Переменные в стековой памяти не нужно удалять вручную. Они автоматически деаллоцируются, когда выходят из области видимости. (Обратитесь к другому уроку по чтению, посвященному области видимости)

В общем, помните, что если Вы используете команду "new", то Вам также потребуется "delete", а после удаления Вы должны установить значение nullptr.






Современный цикл "for", основанный на диапазоне

В последних версиях C++ существует версия цикла for, которая автоматически выполняет итерацию по всем элементам в контейнере. Это очень удобно при использовании контейнера стандартной библиотеки, поскольку Вам не нужно беспокоиться о попытках доступа к памяти, например, за пределами безопасного диапазона: цикл автоматически начнется и закончится в нужном месте.

for ( объявление временной переменной: контейнер ) { тело цикла }

Есть одна важная деталь, касающаяся временной переменной. Если Вы объявляете в цикле обычную временную переменную, то она просто получает по значению копию текущего элемента цикла. Изменения, которые Вы делаете с этой временной копией, не повлияют на реальный контейнер!

#include <iostream>
#include <vector>
int main() {

  // In the standard library, a std::vector is an array with automatic size.
  // Let's make a vector of ints and loop over the contents.
  // The syntax for std::vector<> is discussed further in the lecture on template types.

  std::vector<int> int_list;
  int_list.push_back(1);
  int_list.push_back(2);
  int_list.push_back(3);

  // Automatically loop over each item, one at a time:
  for (int x : int_list) {
    // This version of the loop makes a temporary copy of each
    // list item by value. Since x is a temporary copy,
    // any changes to x do not modify the actual container.
    x = 99;
  }

  for (int x : int_list) {
    std::cout << "This item has value: " << x << std::endl;
  }

  std::cout << "If that worked correctly, you never saw 99!" << std::endl;

  return 0;
}

Если Вы сделаете временную переменную ссылочного типа, Вы сможете действительно изменять текущий элемент контейнера, а не просто получать его копию. Этот модифицированный пример показывает, как это сделать:

#include <iostream>
#include <vector>
int main() {

  std::vector<int> int_list;
  int_list.push_back(1);
  int_list.push_back(2);
  int_list.push_back(3);

  for (int& x : int_list) {
    // This version of the loop will modify each item directly, by reference!
    x = 99;
  }

  for (int x : int_list) {
    std::cout << "This item has value: " << x << std::endl;
  }

  std::cout << "Everything was replaced with 99!" << std::endl;

  return 0;
}

Существуют и более сложные способы использования этого. Например, если Вы выполняете итерацию по большим объектам в контейнере, то даже если Вы не хотите изменять объекты, Вы можете использовать ссылку на константу в качестве типа переменной цикла, чтобы избежать создания временной копии большого объекта, что в противном случае может быть медленным.

#include <iostream>
#include <vector>
int main() {

  std::vector<int> int_list;
  int_list.push_back(1);
  int_list.push_back(2);
  int_list.push_back(3);

  for (const int& x : int_list) {
    // This version uses references, so it doesn't make any temporary copies.
    // However, they are read-only, because they are marked const!
    std::cout << "This item has value: " << x << std::endl;
    // This line would cause an error:
    //x = 99;
  }

  return 0;
}



//

This item has value: 1
This item has value: 2
This item has value: 3






Беззнаковые целочисленные типы
Что такое беззнаковые целые числа?

Иногда в коде C++ Вы встречаете еще один тип целых чисел - "беззнаковые" целые числа, которые не могут представлять отрицательные значения. Вместо этого, беззнаковые целые числа имеют увеличенный верхний диапазон положительных значений по сравнению со знаковыми целыми числами, занимающими столько же памяти. (Знаковые целые числа занимают немного памяти только для того, чтобы иметь возможность представлять отрицательный знак) Беззнаковые целые числа иногда используются в особых случаях для очень эффективного использования памяти; например, в формате хранения данных они могут использоваться для того, чтобы быть более компактными. Однако смешение беззнаковых и знаковых целых чисел в Вашем коде может привести к неожиданным проблемам. Давайте рассмотрим несколько примеров с беззнаковыми целыми числами.




Синтаксис беззнаковых типов

Обычный синтаксис "int" по умолчанию создает подписанный int. Но Вы можете также написать "signed int" или "signed", чтобы получить тот же тип:

1234
int a = 7;// signed int a = 7;// signed a = 7;

Если Вы напишете "unsigned int" или "unsigned", то сможете создать переменную с типом беззнакового целого:

123
unsigned int b = 8;// unsigned b = 8;

Проблемы с беззнаковой арифметикой

Беззнаковые инты не могут представлять отрицательные значения. Вместо этого они жертвуют возможностью представлять отрицательные числа в обмен на более высокий верхний положительный предел диапазона значений, которые они могут представлять, используя то же количество битов, что и сопоставимое знаковое целое число. Однако базовое битовое представление, которое беззнаковые целые числа используют для этих очень больших значений, фактически является тем же самым, что и представление, которое знаковое целое число использует для своего диапазона отрицательных значений. Это означает, что отрицательное знаковое целое может быть переинтерпретировано как очень большое положительное беззнаковое целое, и наоборот. Это может привести к странному поведению, если Вы смешиваете знаковые и беззнаковые инты. Если Вы выполняете арифметические действия между знаковыми и беззнаковыми интами, то Вы можете получить нежелательные результаты, если по логике должны были возникнуть отрицательные значения.

Проблемы со сложением

Сложение с беззнаковыми значениями может быть без проблем, если только Вы не выходите за пределы максимального диапазона. Это 7+8 и показывает 15, как и ожидалось

12
std::cout << (a+b) << std::endl;

Однако Вам следует быть осторожным, если Вы приближаетесь к верхнему пределу для знаковых значений int, даже если Вы используете беззнаковые значения int. Знаковые инты имеют меньшее максимальное значение, поэтому если Вы получите беззнаковый int, превышающий этот предел, а затем приведете его к знаковому int, он будет интерпретирован как отрицательное число, которого Вы не ожидали.

Проблемы с вычитанием

Теперь давайте рассмотрим проблемы, которые могут возникнуть, если Вы будете выполнять вычитание с беззнаковыми интами и пытаться подразумевать отрицательные значения, которые не могут быть представлены. Создадим несколько беззнаковых интов:

123
unsigned int x = 10;unsigned int y = 20;

Теперь запишем (y-x) как 20-10 в беззнаковой арифметике, что приводит к 10, как и ожидалось:

12
std::cout << (y-x) << std::endl;

Напротив, в следующем примере делается попытка записать (x-y), которая равна 10-20, но беззнаковая арифметика не может представить -10, и вместо этого получается очень большое положительное значение. На выходе получается 4294967286, что близко к максимуму для беззнакового 32-битного целого числа.

12
std::cout << (x-y) << std::endl;

Приведение значений обратно к signed int

Если мы явно приведем результат обратно к знаковому целому числу, то, возможно, мы снова получим что-то пригодное для использования. В результате мы получим -10:

12
std::cout << (int)(x-y) << std::endl;

Вы также можете выполнить операцию приведения для преобразования в signed int, просто присвоив беззнаковый результат переменной signed int. Это также дает результат -10:

123
int z = x - y;std::cout << z << std::endl;

Однако приведение результата не всегда является лучшим способом решения этой задачи! Вместо этого, возможно, Вы захотите создать временные рабочие копии беззнаковых значений в виде знаковых типов. Таким образом, Вы сможете выполнять операции как обычно и вручную проверять, является ли значение отрицательным или положительным, в тех случаях, когда оно не должно быть отрицательным. Например, если Вы присваиваете беззнаковое значение int знаковому int, Вы можете проверить, показывает ли знаковый int отрицательное значение. Если оно отрицательное, это означает, что беззнаковое значение было слишком большим, чтобы его можно было точно привести к знаковому. Вот надуманный пример:

12345678
int test_val = (x-y);if (test_val < 0) {  // Note: The standard error stream (cerr) will be displayed in the terminal.  // You can also handle it separately from the standard output stream (cout)  // if you are logging things to files.  std::cerr << "Warning: unsigned value cast to signed int resulted in a negative value" << std::endl;}

Прямое сравнение между знаковыми и беззнаковыми интами также может вызвать проблемы. Следующая строка может выдать предупреждение или ошибку. На данный момент мы ее закомментировали:

12
// std::cout << (a<b) << std::endl;

Размеры контейнеров часто бывают беззнаковыми

Мы часто называем общий класс структуры данных "контейнером". Библиотека стандартных шаблонов C++ (STL) предоставляет множество таких классов, например, std::vector. Многие из этих типов классов имеют переменную-член size(), которая возвращает количество элементов, находящихся в контейнере в данный момент. Важно отметить, что во многих случаях это даст Вам размер в беззнаковом целочисленном типе. (Точный размер байта конкретного беззнакового целого типа может отличаться.) Поэтому, хотя Вы, вероятно, должны стараться избегать использования беззнаковых целых чисел в своем коде, за исключением очень особых случаев, Вы можете столкнуться со случаями, когда Вы сравниваете знаковый int (возможно, счетчик итераций) с размером беззнакового целого числа. Поэтому будьте готовы к безопасной работе с беззнаковыми целыми числами!

Вот пример, в котором компилятор предупредит Вас, что он сравнивает знаковое целое число i с беззнаковым целым числом, возвращаемым функцией size():

12345
std::vector<int> v = {1,2,3,4};for (int i=0; i < v.size(); i++) {    std::cout << v[i] << std::endl;}

Вы можете использовать различные методы приведения, описанные выше, чтобы предупреждение исчезло. Вы можете просто использовать в качестве типа для счетчика i беззнаковое число int или, более того, сам std::vector<int>::size_type. (Однако это может вызвать у Вас больше проблем, если Вы по тем или иным причинам будете выполнять обычные арифметические операции над i, поскольку Вы снова вводите в свой код беззнаковое целое число. Давайте пока предположим, что Вашим контейнерам не потребуется содержать астрономическое количество элементов)

Теперь рассмотрим опасность попытки вычесть из unsigned int, представляющего размер. Если мы напишем код следующим образом, что может пойти не так?

12345
std::vector<int> v = {1,2,3,4};for (int i=0; i <= v.size()-1; i++) {    std::cout << v[i] << std::endl;}

Этот код работает, как и ожидалось, но только потому, что размер ненулевой. Если бы у нас возникла ситуация, в которой этот цикл мог бы обработать пустой вектор, то программа потерпела бы крах:

12345
std::vector<int> v;for (int i=0; i <= v.size()-1; i++) {    std::cout << v[i] << std::endl;}

Здесь вектор v по умолчанию инициализируется пустым, поэтому его size() сообщается как 0. Это означает, что выражение "v.size()-1" будет равно -1 , интерпретируемому как целое число без знака, в данном случае означающее очень большое положительное целое число. Таким образом, на первой итерации цикла выражение "0 <= (очень большое положительное целое число)" будет оценено как true, и тело цикла выполнится, оценивая v[0], несмотря на то, что в v нет первого элемента для доступа. В результате этот код, скорее всего, вызовет segfault и аварийное завершение работы.

Таким образом, здесь наиболее разумно, возможно, написать это способом "i < v.size()", но эти другие приемы также решат эту проблему. Давайте подумаем, почему:

123456789101112
// Casting to signed int first helps to ensure that the result// of subtraction will truly be a signed negative value when size is 0:for (int i=0; i <= (int)v.size()-1; i++) {    // ...} // Rewriting the algebra to perform addition instead of subtraction// helps to avoid going below 0:for (int i=0; i+1 <= v.size(); i++) {    // ...}

На практике, чтобы этот код действительно был полностью надежным, Вам также необходимо проверить i на максимальное ограничение для целого числа данного типа и убедиться, что контейнер никогда не вырастет настолько большим. Однако это уже отдельные проблемы. А пока просто обратите внимание на то, когда Вы выполняете преобразование в беззнаковый тип или из него, и следите за потенциальными проблемами, которые это может вызвать.

Заключение

Будьте осторожны, когда имеете дело с беззнаковыми целыми числами! В целом, Вы должны использовать беззнаковые целые числа только в случае крайней необходимости, например, если Вы пытаетесь максимально расширить диапазон положительных значений, которые Вы можете хранить в одном и том же объеме памяти. Для повседневных целей высокоуровневого программирования Вам могут понадобиться только знаковые целые числа. Если Вам необходимо обеспечить положительные значения во время выполнения кода, Вы можете вписать в код проверки безопасности, которые будут отслеживать значения и выдавать предупреждения или ошибки при превышении определенного диапазона.





ТИПЫ ШАБЛОНОВ

std::vector

Добавление и изменение:
push_back(значение) — положить элемент в конец. Самый частый способ.
pop_back() — выбросить последний элемент (когда передумали).
clear() — полностью опустошить вектор (сделать его размер равным 0).

Информационные:
.size() — узнать, сколько элементов сейчас внутри. Очень важно для циклов!
.empty() — проверить, не пуст ли вектор (возвращает true или false).
.capacity() — узнать, сколько места зарезервировано в памяти «на вырост».

Доступ к данным:
v[i] — быстрый доступ по индексу (как в массиве). Но осторожно: если индекс неверный, будет Segfault!
v.at(i) — безопасный доступ. Если ошибетесь с индексом, программа выдаст понятную ошибку вместо вылета.
