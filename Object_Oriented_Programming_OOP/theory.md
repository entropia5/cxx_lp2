

Основы ООП строятся на:
1. Класс
2. Объект
3. Конструктор/деструктор


Класс - чертёж. Чертёж описывает какие есть данные и какие есть действия.
Шаблон описывающий состояние и поведение объектов которые будут создаваться на его основе.

класс - имя класса {
    //свойства и методы класса
}


class Car
{
public:
    int speed;
    int fuel;

    void drive()
    {
        speed += 10;
    }
};

Class описывает 
данные (переменные) speed, fuel
поведение(ф-ции) drive() 

однако это только описание.

поля — это данные объекта.




Object
- это вещь, созданная по чертежу.
Класс - чертеж данной машины
Объект - это реальная машина.

Car myCar;
myCar - это объект класса Car;

можно создать много....
Car car1;
Car car2;
Car car3;

у каждого будут свои данные.


Конструктор - это функция, которая вызывается при создании объекта.
подготавливает объект к работе.

В нашем примере:
задаёт начальную скорость
задаёт здоровье
задаёт имя


class Player
{
public:

    int health;

    Player()
    {
        health = 100;
    }
};

когда создадим объект Player hero; произойдет автоматически Player(), то есть health = 100;




Конструктор с параметрами
Можно передать значения.


class Player
{
public:

    int health;

    Player(int h)
    {
        health = h;
    }
};



Player hero(150);


Теперь hero.health = 150;





Деструктор — функция, которая вызывается когда объект уничтожается

освобождает память
закрывает файл
удаляет ресурсы



class Player
{
public:

    Player()
    {
        cout << "Player created\n";
    }

    ~Player()
    {
        cout << "Player destroyed\n";
    }



};



Схематично:

class Player   → описание
Player hero    → объект
Player()       → конструктор
~Player()      → деструктор



объекты живут по области видимости (scope).Создал в блоке { } → уничтожится при выходе из блока.


#include <iostream>
using namespace std;

class Player
{
public:

    int health;

    Player()
    {
        health = 100;
        cout << "Player created\n";
    }

    ~Player()
    {
        cout << "Player destroyed\n";
    }
};

int main()
{
    Player hero;

    cout << hero.health;

    return 0;
}



1. ОС вызывает main()
2. создаётся объект hero
3. вызывается Player()
4. выводится hero.health
5. программа возвращает 0
6. объект hero уничтожается
7. вызывается ~Player()


Player hero; уничтожается автоматически после выхода из main; поэтому вызывается ~Player()

main начинается
      ↓
Player hero
      ↓
конструктор Player()
      ↓
объект используется
      ↓
main заканчивается
      ↓
деструктор ~Player()




Память.
STACK  - быстрая память для локальных переменных
HEAP   - память для динамических объектов

int main()
{
    Player hero;
}

Тут объект создаётся в стеке (stack memory).



Когда компилятор видит:
Player hero;

он делает 3 шага:
1. Выделяет память...Размер равен сумме всех полей...sizeof(Player)
2. Запускает конструктор...Вызывается Player()....
Player()
{
    health = 100;
    armor = 25;
}


3. Объект готов к использованию....можно писать:
hero.health
hero.armor




cout << hero.health;
Здесь:

1. находит объект hero
2. находит поле health
3. читает значение
4. отправляет в cout


Когда выходим из блока:
{
    Player hero;
}

то:

1. вызывается деструктор
2. память освобождается


Вызывается ~Player()





main()
{
    Player hero;
}

запуск программы
      ↓
вход в main
      ↓
выделение памяти под hero
      ↓
вызов Player()
      ↓
работа с объектом
      ↓
конец main
      ↓
вызов ~Player()
      ↓
освобождение памяти
