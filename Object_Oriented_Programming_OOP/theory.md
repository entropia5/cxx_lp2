Конструктор - это функция, которая вызывается при создании объекта.
подготавливает объект к работе.

В нашем примере:
задаёт начальную скорость
задаёт здоровье
задаёт имя


class Player
{
public:

    int health;

    Player()
    {
        health = 100;
    }
};

когда создадим объект Player hero; произойдет автоматически Player(), то есть health = 100;




Конструктор с параметрами
Можно передать значения.


class Player
{
public:

    int health;

    Player(int h)
    {
        health = h;
    }
};



Player hero(150);


Теперь hero.health = 150;





Деструктор — функция, которая вызывается когда объект уничтожается

освобождает память
закрывает файл
удаляет ресурсы



class Player
{
public:

    Player()
    {
        cout << "Player created\n";
    }

    ~Player()
    {
        cout << "Player destroyed\n";
    }



};



Схематично:

class Player   → описание
Player hero    → объект
Player()       → конструктор
~Player()      → деструктор



объекты живут по области видимости (scope).Создал в блоке { } → уничтожится при выходе из блока.


#include <iostream>
using namespace std;

class Player
{
public:

    int health;

    Player()
    {
        health = 100;
        cout << "Player created\n";
    }

    ~Player()
    {
        cout << "Player destroyed\n";
    }
};

int main()
{
    Player hero;

    cout << hero.health;

    return 0;
}



1. ОС вызывает main()
2. создаётся объект hero
3. вызывается Player()
4. выводится hero.health
5. программа возвращает 0
6. объект hero уничтожается
7. вызывается ~Player()


Player hero; уничтожается автоматически после выхода из main; поэтому вызывается ~Player()

main начинается
      ↓
Player hero
      ↓
конструктор Player()
      ↓
объект используется
      ↓
main заканчивается
      ↓
деструктор ~Player()

static 
внутри класса

Когда static используется в классе, это означает:
Поле или функция принадлежат всему классу, а не отдельному объекту.

Обычное поле — у каждой машины своё.
static поле — одно на все машины.



class Car {
public:
    static int count;   // одно поле на все объекты

    Car() {
        count++;
    }
};

int Car::count = 0;



//Использование

Car a;
Car b;

cout << Car::count;  // 2



//Здесь

a.count ❌
b.count ❌
Car::count ✅

Потому что поле принадлежит классу.




2. static функция класса
Функция тоже может быть static.

class Car {
public:
    static void print() {
        cout << "Hello";
    }
};


//вызываем

Car::print();


static функция не видит обычные поля объекта.

потому что нет объекта.
например
class Car {
    int speed;

    static void f() {
        cout << speed; // ❌ нельзя
    }
};




static функция работает без экземпляра класса.



3. static переменная внутри функции
Это совсем другое значение.

Такая переменная:
создаётся один раз
сохраняет значение между вызовами функции

void counter() {
    static int x = 0;
    x++;
    cout << x << endl;
}


//вызываем
counter(); // 1
counter(); // 2
counter(); // 3

Без static было бы всегда 1.





4. static в глобальной области
Это используется для ограничения видимости внутри файла.


static int x = 10;



    Где используется	        Что делает

static в классе	            одно поле на весь класс
static функция	            вызывается без объекта
static в функции	        сохраняет значение между вызовами
static глобально	        видна только внутри файла





в ООП:
static — это член класса, который принадлежит всему классу, а не конкретному объекту.

То есть: 
обычные поля → у каждого объекта свои
static поля → одно общее для всех объектов
