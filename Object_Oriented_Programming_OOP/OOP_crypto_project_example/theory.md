ВИРТУАЛЬНЫЕ Ф-ЦИИ И ПОЛИМОРФИЗМ.
- это позволяет нам работать с объектами производных классов через указатель на базовый класс.

Виртуальные функции — помечаем virtual в базовом классе, потом override в производном.




Полиморфизм позволяет одному интерфейсу управлять разным поведением объектов.
- у нас есть указатель на базовый класс, но мы хотим, чтобы вызов метода вёл себя как у потомка.


без виртуал ф-ций:
Base* obj = new Derived();
obj->show();  // вызовет Base::show, а не Derived::show


с виртуал ф-циями:

class Base {
public:
    virtual void show() { std::cout << "Base\n"; }
};

class Derived : public Base {
public:
    void show() override { std::cout << "Derived\n"; }
};

Base* obj = new Derived();
obj->show();  // теперь выведет "Derived"






полиморфизм - одно «имя функции» → разные реализации.







class Base {
protected:
    int x;
public:
    Base(int val) : x(val) {}  // конструктор
    virtual ~Base() {}          // виртуальный деструктор!
    virtual void show() {       // виртуальная функция
        std::cout << "Base: " << x << "\n";
    }
};

class Derived : public Base {
private:
    int y;
public:
    Derived(int a, int b) : Base(a), y(b) {}
    void show() override {      // переопределение виртуальной функции
        std::cout << "Derived: " << x << ", " << y << "\n";
    }
};







Абстрактные классы и интерфейсы


Виртуальный метод — у базового класса есть реализация, но потомки могут её переопределить


Чисто виртуальный метод — у базового класса нет нормальной реализации для использования, и потомки обязаны его реализовать.
